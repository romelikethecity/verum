Format vs Reality

john.smith@acme.com looks like a valid email address. Correct format, plausible domain, nothing obviously wrong. But John Smith left Acme two years ago. The mailbox doesn't exist anymore. Send to this address and it bounces.

Format validation catches addresses that are obviously broken: missing @ symbols, double dots, invalid characters. But it can't tell you whether the mailbox actually exists. For that, you need to ask the mail server.

That's what real email validation does. We don't just check format. We check that the domain exists, that it has mail servers configured, and that the specific mailbox is accepting mail. The difference between "looks valid" and "is valid" can be the difference between a successful campaign and a deliverability disaster.

How We Validate

  • Syntax validation. Is the address properly formed? Correct characters, proper format, no obvious errors.
  • DNS lookup. Does the domain exist? A valid-looking address at a defunct domain will never deliver.
  • MX record check. Does the domain have mail servers configured? Some domains exist but don't receive email.
  • SMTP verification. Does this specific mailbox exist? We ask the mail server directly without sending a message.
  • Catch-all detection. Some domains accept all mail regardless of mailbox existence. We identify these separately.
  • Disposable domain detection. Temporary email services like Mailinator. Often used for fake signups.
  • Role address identification. Generic addresses like info@, support@, admin@. Often have different engagement patterns.

The Validation Sequence

Each check catches different types of invalid addresses:

Step 1: Syntax. Catches typos and formatting errors. "john.smith@acme" fails (no TLD). "john..smith@acme.com" fails (double dot).

Step 2: Domain. Catches addresses at defunct or fake domains. "john@acmecorp-fake.com" fails if that domain doesn't exist.

Step 3: MX records. Catches addresses at domains that exist but don't receive mail. Some companies have websites but no email.

Step 4: Mailbox. Catches addresses where the domain is fine but the specific user doesn't exist. John left, his mailbox was deleted.

An address must pass all four steps to be considered valid.

Common Questions

How does validation work?

Syntax check, DNS lookup, MX record check, SMTP verification. Each step catches different invalid addresses.

Do you send test emails?

No. SMTP verification confirms mailbox existence without sending messages. We ask the server if the address exists, then disconnect.

Validation vs verification?

Terms are often interchangeable. Technically, validation checks format; verification confirms existence. We do both.

Why not just use regex?

Regex only checks format. "fake@doesnotexist.com" passes regex but bounces. Real validation checks that the mailbox exists.
